fastdfs安装

FastDFS是一个开源的分布式文件系统，她对文件进行管理，功能包括：文件存储、文件同步、文件访问（文件上传、文件下载）等，解决了大容量存储和负载均衡的问题。特别适合以文件为载体的在线服务，如相册网站、视频网站等等。

# 上传安装介质

本次安装以源码进行安装需要下载以下文件：

  1. fastdfs-master.zip (opens new window)
  2. libfastcommon.git (opens new window)

上传安装文件到服务器的/data目录

本次安装的服务器有

  1. ..*.11
  2. ..*.12
  3. ..*.13

# 开放端口

 firewall-cmd --add-port=22122/tcp --permanent &&  firewall-cmd --add-port=23000/tcp --permanent && firewall-cmd --reload
1

# 解压安装

yum install unzip -y
unzip /data/libfastcommon-master.zip -d /data/fastdfs
unzip /data/fastdfs-master.zip -d /data/fastdfs
cd /data/fastdfs/libfastcommon-master/ && ./make.sh && ./make.sh install 
cd /data/fastdfs/fastdfs-master/ && ./make.sh && ./make.sh install
1

# 创建文件夹

mkdir -p /data/fastdfs/data/tracker
mkdir -p /data/fastdfs/data/storage

# 拷贝配置文件

cp /etc/fdfs/tracker.conf.sample /etc/fdfs/tracker.conf && cp /etc/fdfs/storage.conf.sample /etc/fdfs/storage.conf && cp /etc/fdfs/client.conf.sample /etc/fdfs/client.conf
1
2

# 配置tracker

修改配置文件*/etc/fdfs/tracker.conf* 修改位置有

  1. 22行：base_path=/data/fastdfs/data/tracker
  2. 54行：store_group=group
  3. 98行：run_by_group=root
  4. 102行：run_by_user=root

# 配置storage

修改配置文件*/etc/fdfs/storage.conf* 修改位置有

  1. 11行：group_name=group
  2. 41行：base_path=/data/fastdfs/data/storage
  3. 111行：store_path0=/data/fastdfs/data/storage
  4. 119行：tracker_server=*.*.*.11:22122,*.*.*.12:22122,*.*.*.13:22122
  5. 134行：run_by_group=root
  6. 138行：run_by_user=root

# 启动

/usr/bin/fdfs_trackerd /etc/fdfs/tracker.conf start && /usr/bin/fdfs_storaged /etc/fdfs/storage.conf start
1
2

# 配置客户端

fdfs自带了一个客户端用户查看集群状态，测试上传等功能，首先配置好/etc/fdfs/client.conf配置,修改其中的tracker的地址即可

# 查看集群状态

在*...11服务器上查看状态

fdfs_monitor /etc/fdfs/storage.conf
1

# 查看storage状态

fdfs_monitor /etc/fdfs/client.conf
1

# 测试上传

使用客户端上传命令为

/usr/bin/fdfs_test /etc/fdfs/client.conf upload Burner.jpg
1

# 结束语

fastdfs整体结构简单来说就是如下，tracker管理storage状态，storage进行存储，所以在上传时需要首先从tracker获取storage的地址，然后链接storage进行上传文件，fsatdfs还包含了一个文件访问模块，基于Nginx实现的一个文件访问，该模块可以做到整个集群的文件访问，如果某些storage的文件有缺失，只要有storage的文件是完整的都不影响访问，本次只使用了文件存储，没有使用文件访问故没有安装该模块。
