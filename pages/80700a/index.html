Docker运行微服务

微服务作为一个架构天然包含了许多进程模块，这是微服务的设计理念，将服务拆分成合适的粒度运行在不同的进程中，这给部署带来了一定的困难，好在容器的流行给部署带来了很大的简化，在开发、测试等非正式环境可能没有条件或者足够的资源去运行整个k8s集群，此时可以选择将微服务部署在同一个主机的Docker里，本文就讲述了如何在同一主机的Docker里部署微服务系统

# 说明

本次用例均在同一个主机里面运行，微服务架构基于SpringCloud,注册中心采用Euraka，在Docker中部署微服务主要要解决的问题的时服务之间通信的问题，因为Docker容器是相互隔离的，默认情况不能进行通信，Docker容器间通信方式大概有以下几种

  1. 通过Docker为容器分配的ip进行通信
  2. 通过link的方式进行容器之间连接来达到通信的方式
  3. 通过同一个network网络来通信

通过ip的方式会导致微服务间通信有不确定性，因为每次容器启动ip可能都会变动；而通过link的方式则会增加部署的难度；故采用network的方式来达到容器通信的目的，本次运行的服务包含以下

  1. eureka
  2. zuul
  3. app1
  4. app2
  5. app3

# 创建bridge网络

docker默认的网络就是bridge网络，但它和我们自己创建的有一定的区别，具体区别可以参考https://docs.docker.com/network/bridge/#differences-between-user-defined-bridges-and-the-default-bridge (opens new window)

运行以下命令创建网络

docker network create --driver bridge appnet
1

可以查看刚才创建的网络

docker network ls
1

# 修改注册中心地址

将各个模块的注册中心地址修改为以下值即可

eureka.client.serviceUrl.defaultZone=http://eureka:8763/eureka
1

# 添加容器参数

在启动eureka容器时添加*--network和--network-alias*参数

docker run -d --name eureka --network appnet --network-alias eureka -p 8763:8763  eureka
1

所有模块的启动命令就会成为下面的样子，整个微服务其实只需要暴露网关zuul的端口即可，暴露eureka的端口是为了方便查看服务在线情况

docker run -d --name eureka --network appnet --network-alias eureka -p 8763:8763  eureka
docker run -d --name zuul --network appnet --network-alias zuul -p 8000:8000  zuul
docker run -d --name app1 --network appnet --network-alias app1 app1
docker run -d --name app2 --network appnet --network-alias app2 app2
docker run -d --name app3 --network appnet --network-alias app3 app3

# 脚本

为了避免频繁输入命令，现提供一个启动的脚本和批量启动的脚本

# 思路

启动一个模块的流程应是如下流程

# 脚本内容

单个脚本名称：start-app.sh

#!/bin/bash
set -u
set -e
name=$1

echo "检查有没有创建名称为appnet网络"
appnet=$(docker network ls|grep appnet)

if [ ! -n "$appnet" ];then
  echo "没有初始化名称为appnet的网络，开始初始化"
  docker network create --driver bridge appnet
fi

echo "检查是否有$name 服务的容器"
cid=$(docker ps -a |grep $name | awk '{print $1}')

if [ -n "$cid" ];then
  echo "删除$name 服务的容器"
  docker rm -v -f $cid
fi

echo "启动$name 服务"

extend=""

if [ "$name" = "eureka" ];then
  extend="-p 8763:8763"
elif [ "$name" = "zuul" ];then
  extend="-p 8000:8000"
fi

docker run -d --name $name --network appnet --network-alias $name $extend  $name
