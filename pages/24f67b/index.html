Nginx安装

Nginx作为高性能的HTTP和反向代理web服务器，在前后端分离的模式中承担着重要的角色，本文将从源码安装和Docker安装两个方面，介绍Nginx的安装流程

# 源码安装

# 上传安装包

安装包下载路径：http://nginx.org/en/download.html (opens new window) 本次安装版本1.19.5

下载好后上传到服务器的/data目录

需要安装的服务器有：

  1. ..*.11
  2. ..*.12
  3. ..*.13

# 安装

安装过程中如缺少 什么依赖会在*./configure*阶段进行说明，到时候根据缺少的安装

yum update -y # 系统缺少包需要先升级
cd /data
tar -xzvf nginx-1.19.5.tar.gz && mkdir /data/nginx 
yum install -y pcre-devel openssl-devel
cd /data/nginx-1.19.5 && \
   ./configure --prefix=/data/nginx/ --sbin-path=/data/nginx/nginx --conf-path=/data/nginx/nginx.conf --pid-path=/data/nginx/nginx.pid --with-http_ssl_module  --with-stream --with-stream_ssl_module --with-http_stub_status_module
make && make install

# Docker安装

# 说明

官方nginx镜像的默认路径

默认conf挂载目录: /etc/nginx/conf.d/ 和 /etc/nginx/nginx.conf，该配置会引用/etc/nginx/conf.d/default.conf文件，一般来说我们修改这个文件下面的内容即可

默认的html挂载目录：/usr/share/nginx/html

默认的log目录：/var/log/nginx

# 拉取镜像

# 搜索，此处不是必须
docker search nginx
docker pull nginx
1
2

# 创建compose文件

mkdir -p /u01/componse/nginx
vim /u01/componse/nginx/docker-compose.yml
## 以下是内容
nginx:
    image: nginx
    restart: always
    hostname: 127.0.0.1
    ports:
        - 80:80
        - 443:443
    volumes:
        - ./conf/conf.d:/etc/nginx/conf.d
        - ./html:/usr/share/nginx/html
        - ./log:/var/log/nginx
        # 证书目录
        - /u01/ssl:/ssl
