Nginx常用配置

作为前后端分离的连接点，Nginx通常的配置有正向代理，反向代理，https等，现将常用配置记录如下

# 反向代理

反向代理会保留请求者的信息，在前后端分离时经常使用，并且会结合负载均衡一起使用。

在http模块下进行如下配置

http {
    include       mime.types;
    default_type  application/octet-stream;
    # 后台网关服务
    upstream apiServer {
        # 负载均衡 max_fails为健康检查的最大失败次数
        server 192.168.1.3:8000 weight=1 max_fails=3 fail_timeout=30s;
        server 192.168.1.4:8000 weight=1 max_fails=3 fail_timeout=30s;
        server 192.168.1.5:8000 weight=1 max_fails=3 fail_timeout=30s;
    }
    sendfile        on;
    keepalive_timeout  65;
    gzip  on;
    server {
        listen       80;
        server_name  localhost;
        #charset koi8-r;
        #access_log  logs/host.access.log  main;
        add_header strict-transport-security max-age=15552000;
        add_header x-content-type-options nosniff;
        add_header x-dns-prefetch-control off;
        add_header x-download-options noopen;
        add_header x-frame-options SAMEORIGIN;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        location / {
            root   html;
            index  index.html index.htm;
        }
        location /api/ {
            proxy_pass http://apiServer/;
            proxy_redirect     off;
            proxy_set_header   Host             $http_host;
            proxy_set_header   X-Real-IP        $remote_addr;
            proxy_set_header   X-Forwarded-For  $proxy_add_x_forwarded_for;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
       }
        error_page   500 502 503 504  /50x.html;
        location = /50x.html {
            root   html;
        }
    }
}

# 正向代理

正向代理会影响请求者的信息，对被访问的服务器来说，访问者就是nginx所在的服务器，可用于内外服务器借助于Nginx访问外部接口服务

location / {
  proxy_pass https://aip.baidubce.com;
  proxy_set_header Host aip.baidubce.com;
  proxy_set_header X-Real-IP $remote_addr;
  proxy_set_header REMOTE-HOST $remote_addr;
  add_header X-Cache $upstream_cache_status;
  add_header Cache-Ctontrol no-cache;
}

# https证书配置

作为web项目对外提供服务希望保证安全都会尽量采用https的形式访问，作为微服务架构来说内部会有很多系统，这些系统之间经常互相访问，这个时候采用https就不太合适，一般的做法时将Nginx作为前端访问的入口，在此处添加https，而Nginx到后端系统作为内网访问直接以http的形式进行访问，这样对整个系统都没有影响，不需要将后端所有模块都添加https的支持，具体配置如下

server {
  listen       443 ssl;
  server_name  test.com.cn;
  # 证书pem路径
  ssl_certificate      /data/nginx/ssl/test.pem;
  # 证书key的路径
  ssl_certificate_key   /data/nginx/ssl/test.key;
  ssl_session_cache    shared:SSL:1m;
  ssl_session_timeout  5m;
  ssl_protocols TLSv1 TLSv1.1 TLSv1.2;
  ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:ECDHE:ECDH:AES:HIGH:!NULL:!aNULL:!MD5:!ADH:!RC4;
  ssl_prefer_server_ciphers  on;
  add_header strict-transport-security max-age=15552000;
  add_header x-content-type-options nosniff;
  add_header x-dns-prefetch-control off;
  add_header x-download-options noopen;
  add_header x-frame-options SAMEORIGIN;
  proxy_set_header Upgrade $http_upgrade;
  proxy_set_header Connection "upgrade";

  location / {
      root   html;
      index  index.html index.htm;
  }
}

# http转https

作为客户访问时，一般来说不会刻意去记你的网站是否是支持https的，可能会以http的形式访问你的网站，此时需要自动跳转到https的访问，可在Nginx作如下配置

在server中进行重定向

  server {
      listen       80;
      rewrite ^(.*) https://$server_name$1 permanent;
  }

在http模块中进行重定向

如果是http直接重定向

  if ($scheme = http ) {
    return 301 https://$host$request_uri;
  }

如果是80端口就重定向

  if ($server_port = 80 ) {
    return 301 https://$host$request_uri;

判断不是https协议就跳转，以及判断不是指定域名就跳转到指定域名

  if ($ssl_protocol = "") { return 301 https://$server_name$request_uri; }
  if ($host != www.xxxx.com) { return 301 $scheme://www.xxxx.com$request_uri; }

# 防止域名恶意解析

有些时候会遇到有人将自己的域名解析到你的ip上的情况，导致访问他的域名最终访问的是你的资源，这个时候可以在Nginx上限制只能指定遇到才能进行解析访问

  if ($host != www.xxxx.com) { return 444; }
